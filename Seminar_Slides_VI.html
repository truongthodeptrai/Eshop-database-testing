Cơ sở dữ liệu NoSQL

NoSQL chuyển nhiều trách nhiệm hơn sang kiểm thử ứng dụng

{
  "_id": "order-1042",
  "userId": "user-7",
  "items": [
    { "sku": "P12", "qty": 2 }
  ],
  "total": 450000,
  "status": "pending",
  "schemaVersion": 3
}
  • Cấu trúc document, field lồng nhau và mảng
  • Reference do ứng dụng quản lý
  • Truy vấn và aggregation pipeline
  • Nhất quán mạnh hoặc nhất quán cuối cùng
  • Partition key, shard và hot partition
  • Replication, retry và idempotency
  • Nhiều phiên bản document cùng tồn tại

Không dùng sleep cố định: hãy retry có giới hạn và đặt timeout rõ ràng cho eventual consistency.
