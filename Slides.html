Why it matters

Database bugs can survive a successful UI flow

UI

Visible result

The screen reports success, but it cannot prove every persisted field is correct.

DB

Stored truth

Rows, documents, relationships, totals, and status values reveal the real outcome.

Test

Repeatability

A controlled starting state makes failures reproducible and easier to debug.

A passing interface flow is evidence. A database assertion is verification.

Database Rider
